## 1. What Is This SDK?

This SDK lets a **React Native app** open a **Flutter-powered chat screen** as a full-screen native Android Activity. 

The idea is:
- You write the chat UI in Flutter (beautiful, fast, portable)
- You package it as a native Android library (AAR)
- A thin React Native bridge exposes a single JavaScript function `openFlutterChat()`
- The consumer React Native app calls that function and the Flutter screen opens natively

**Key facts:**
- Android only (iOS not supported yet)
- Distributed via GitHub URL (no npm registry)
- Version pinned via git tags (e.g. `v1.0.21`)
- One function call from JS to open the entire Flutter screen

### What Each Key File Does

| File | Purpose |
|------|---------|
| `package.json` (root) | Makes `npm install github:...` work. npm always looks for `package.json` at repo root |
| `react-native.config.js` (root) | Tells React Native autolinking where to find the Android source folder |
| `setup.js` (root) | Runs automatically after `npm install`. Patches the consumer app's Android files |
| `publish-sdk.yml` | GitHub Actions workflow. Builds the AAR and creates a GitHub Release |
| `chat_screen.dart` | The Flutter chat UI. This is the screen users see when chat opens |
| `platform_channel.dart` | Bridge between Flutter (Dart) and Kotlin using Flutter's MethodChannel API |
| `FlutterChatModule.kt` | The React Native Native Module. Receives `openFlutterChat()` calls from JavaScript |
| `FlutterChatActivity.kt` | Android Activity that starts and hosts the Flutter engine |
| `flutter_release-1.aar` | Pre-built binary of the Flutter chat module. Committed to git by CI |
| `flutter_repo/` | Maven repository format of the same AAR. Required by Gradle for dependency resolution |

## 3. How Everything Works — End to End

### Full Data Flow: JavaScript → Flutter Screen

```
React Native App (JavaScript)
        │
        │  openFlutterChat({ id, name, email })
        │
        ▼
   index.ts (JS Bridge)
        │
        │  NativeModules.FlutterChatModule.openChat(...)
        │
        ▼
   FlutterChatModule.kt (Kotlin Native Module)
        │
        │  Creates Android Intent with user data as extras
        │  Calls activity.startActivity(intent)
        │
        ▼
   FlutterChatActivity.kt (Android Activity)
        │
        │  Extends Flutter's FlutterActivity
        │  Sets up a MethodChannel named "com.flutterchat.sdk/bridge"
        │  Waits for Flutter to signal "flutterReady"
        │
        ▼
   Flutter Engine starts
        │
        │  main.dart runs
        │  chat_screen.dart mounts
        │  platform_channel.dart calls signalReady()
        │
        ▼
   FlutterChatActivity receives "flutterReady"
        │
        │  Sends user data back via "initData" method call
        │
        ▼
   chat_screen.dart receives user data
        │
        │  Displays: "Chat Support - development build"
        │  Shows user name and email in AppBar
        │  Loads demo messages
        │  Starts 5-second debug timer (prints to logcat)
        │
        ▼
   User sees Flutter chat screen
```

### How the AAR Gets Into the App

```
flutter_chat_module/  (Flutter source code)
        │
        │  flutter build aar --no-profile --no-debug --build-number 1
        │
        ▼
  build/host/outputs/repo/  (Maven repository with AAR)
        │
        │  CI copies files to:
        │
        ▼
  react-native-flutter-chat/android/libs/
        ├── flutter_release-1.aar     (flat copy)
        └── flutter_repo/             (full Maven repo)
        │
        │  Committed to GitHub
        │
        ▼
  Consumer runs: npm install github:neerajj911/flutter-chat-sdk#v1.0.21
        │
        │  npm downloads the repo at that tag
        │  setup.js runs (patches Android files)
        │
        ▼
  android/build.gradle reads:
        implementation 'com.flutterchat.module:flutter_release:1'
        maven { url uri("$projectDir/libs/flutter_repo") }
        │
        ▼
  Gradle finds the AAR inside flutter_repo/
  Links it into the consumer app's build
```

### MethodChannel Communication (Flutter ↔ Kotlin)

The channel name is `"com.flutterchat.sdk/bridge"`. Both Flutter and Kotlin must use the exact same string.

```
Flutter side (platform_channel.dart):
  signalReady()  →  calls method "flutterReady"  →  Kotlin receives it
  
Kotlin side (FlutterChatActivity.kt):
  receives "flutterReady"  →  calls method "initData" with user map  →  Flutter receives it
  
Flutter side (chat_screen.dart):
  receives "initData"  →  builds ChatUser object  →  renders screen with user data
```

## 4. How to Integrate the SDK Into Any React Native App

### Step 1 — Install the SDK

```bash
# In your React Native project root:
npm install github:neerajj911/flutter-chat-sdk#v1.0.21
```

> **Important for future updates:** Always delete the old install first to avoid npm cache issues:
> ```bash
> Remove-Item -Recurse -Force node_modules\react-native-flutter-chat
> npm install github:neerajj911/flutter-chat-sdk#v1.0.XX --force
> ```

### Step 2 — Verify setup.js ran

After `npm install`, check that these changes were made automatically:

**`android/build.gradle`** should have these maven repos added:
```groovy
allprojects {
    repositories {
        maven { url uri("path/to/flutter_repo") }
        maven { url "https://storage.googleapis.com/download.flutter.io" }
    }
}
```

**`android/app/build.gradle`** should have:
```groovy
defaultConfig {
    minSdkVersion 24   // Flutter requires minimum 24
}
```

**`android/app/src/main/AndroidManifest.xml`** should have:
```xml
<activity
    android:name="com.flutterchat.sdk.FlutterChatActivity"
    android:configChanges="orientation|keyboardHidden|keyboard|screenSize|locale|layoutDirection|fontScale|screenLayout|density|uiMode"
    android:hardwareAccelerated="true"
    android:windowSoftInputMode="adjustResize"
    tools:replace="android:windowSoftInputMode" />
```

**`android/settings.gradle`** should have:
```groovy
include ':react-native-flutter-chat'
project(':react-native-flutter-chat').projectDir = new File(...)
```

### Step 3 — Use in your App.tsx / any screen

```typescript
import React, { useState } from 'react';
import {
  View, Text, TextInput, TouchableOpacity,
  StyleSheet, SafeAreaView
} from 'react-native';
import { openFlutterChat } from 'react-native-flutter-chat';

export default function App() {
  const [userId, setUserId]   = useState('user123');
  const [name, setName]       = useState('John Doe');
  const [email, setEmail]     = useState('john@example.com');

  const handleOpenChat = () => {
    openFlutterChat({
      id:    userId,
      name:  name,
      email: email,
    });
  };

  return (
    <SafeAreaView style={styles.container}>
      <Text style={styles.title}>My App</Text>

      <TextInput
        style={styles.input}
        value={userId}
        onChangeText={setUserId}
        placeholder="User ID"
      />
      <TextInput
        style={styles.input}
        value={name}
        onChangeText={setName}
        placeholder="Name"
      />
      <TextInput
        style={styles.input}
        value={email}
        onChangeText={setEmail}
        placeholder="Email"
      />

      <TouchableOpacity style={styles.button} onPress={handleOpenChat}>
        <Text style={styles.buttonText}>Open Chat</Text>
      </TouchableOpacity>
    </SafeAreaView>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, padding: 20 },
  title:     { fontSize: 24, fontWeight: 'bold', marginBottom: 20 },
  input:     { borderWidth: 1, borderColor: '#ccc', borderRadius: 8,
               padding: 10, marginBottom: 12 },
  button:    { backgroundColor: '#0066FF', padding: 14, borderRadius: 8,
               alignItems: 'center' },
  buttonText:{ color: 'white', fontSize: 16, fontWeight: 'bold' },
});
```

### Step 4 — Build and Run

```bash
npx react-native run-android
```

## 6. How to Release a New Version

After committing all your code changes and pushing to GitHub, trigger a new release via GitHub Actions:

### Step 1 — Go to GitHub Actions

Open: `https://github.com/neerajj911/flutter-chat-sdk/actions`

### Step 2 — Run the "Release SDK" Workflow

1. Click **"Release SDK"** in the left sidebar
2. Click **"Run workflow"** (top right)
3. Enter the version tag, e.g. `v1.0.22`
4. Click **"Run workflow"**

### Step 3 — What CI does automatically

The workflow does all of this without you doing anything:

```
1. Checks out the repo
2. Sets up Java 17 and Flutter 3.22.0
3. Runs: flutter pub get
4. Runs: flutter build aar --no-profile --no-debug --build-number 1
5. Copies the built AAR into react-native-flutter-chat/android/libs/
6. Updates version in package.json to match your input (e.g. 1.0.22)
7. git commit -m "chore: release v1.0.22 [skip ci]"
8. git tag v1.0.22
9. git push + push tag
10. Creates a GitHub Release at tag v1.0.22
```

### Step 4 — Verify the release

Go to: `https://github.com/neerajj911/flutter-chat-sdk/releases`

You should see the new release listed with the tag you entered.

## 7. How Consumers Update to a New Version

When you release `v1.0.22`, anyone using your SDK must:

### Option A — Force reinstall (recommended, avoids npm cache issues)

```bash
cd <their-react-native-project>

# 1. Delete the old installed version
Remove-Item -Recurse -Force node_modules\react-native-flutter-chat   # Windows
# OR on Mac/Linux:
rm -rf node_modules/react-native-flutter-chat

# 2. Update package.json manually:
#    Change: "react-native-flutter-chat": "github:neerajj911/flutter-chat-sdk#v1.0.21"
#    To:     "react-native-flutter-chat": "github:neerajj911/flutter-chat-sdk#v1.0.22"

# 3. Install fresh
npm install --force

# 4. Clean Gradle cache and rebuild
cd android
.\gradlew clean           # Windows
./gradlew clean           # Mac/Linux
cd ..

# 5. Run the app
npx react-native run-android
```

### Option B — Direct install with force

```bash
npm install github:neerajj911/flutter-chat-sdk#v1.0.22 --force
cd android && .\gradlew clean && cd ..
npx react-native run-android
```

### Why `--force` and `gradlew clean`?

- **`--force`**: npm caches GitHub packages. Without `--force`, npm uses the cache and you get the old AAR even though you changed the version tag.
- **`gradlew clean`**: Gradle also caches compiled classes. If the AAR changed, Gradle won't pick it up until you force a clean build.

## 8. GitHub Actions Workflow — Deep Dive

File location: `.github/workflows/publish-sdk.yml`

### Why is `.github/` at the repo root and not inside `flutter-chat-sdk/` subfolder?

GitHub Actions **only reads workflows from `.github/workflows/` at the repository root**. If the workflows folder is inside a subfolder, GitHub ignores it completely. That's why the `.github/` folder must be at the top level even though all the actual code is inside `flutter-chat-sdk/`.

```
Repository root  ← GitHub reads workflows from HERE
├── .github/
│   └── workflows/
│       └── publish-sdk.yml   ← GitHub sees this ✔
└── flutter-chat-sdk/
    ├── flutter_chat_module/  ← Flutter code built by the workflow
    └── react-native-flutter-chat/  ← AAR destination
```

### Workflow Trigger

The workflow is triggered **manually** (not on every push). This gives you full control over versions.

```yaml
on:
  workflow_dispatch:
    inputs:
      version:
        description: 'Version tag to create (e.g. v1.2.0)'
        required: true
```

### Key Workflow Steps Annotated

```yaml
# Step 3: Flutter commands run in the subfolder, not the repo root
- name: Flutter pub get
  working-directory: flutter-chat-sdk/flutter_chat_module
  run: flutter pub get

# Step 4: Build the AAR (release mode only, no debug/profile)
- name: Build Flutter AAR
  working-directory: flutter-chat-sdk/flutter_chat_module
  run: flutter build aar --no-profile --no-debug --build-number 1
  # --build-number 1 means the Maven artifact version = 1
  # This must match: implementation 'com.flutterchat.module:flutter_release:1'

# Step 5: Copy AAR to the SDK's android/libs/
- name: Copy AARs to SDK libs
  run: |
    LIBS="flutter-chat-sdk/react-native-flutter-chat/android/libs"
    find flutter-chat-sdk/flutter_chat_module/build/host/outputs/repo \
      -name "*.aar" -exec cp {} "$LIBS/" \;
    cp -r flutter-chat-sdk/flutter_chat_module/build/host/outputs/repo \
      "$LIBS/flutter_repo"

# Step 6: Update version number in package.json
- name: Set version in package.json
  working-directory: flutter-chat-sdk/react-native-flutter-chat
  run: |
    VERSION="v1.0.22"
    PKG_VERSION="${VERSION#v}"   # strips the 'v' → "1.0.22"
    npm version "$PKG_VERSION" --no-git-tag-version

# Step 7: Commit the built AAR + create git tag
- name: Commit and tag
  run: |
    git add .
    git commit -m "chore: release v1.0.22 [skip ci]"
    # [skip ci] prevents another workflow run from triggering
    git tag v1.0.22
    git push && git push --tags
```

### GitHub PAT (Personal Access Token)

The workflow uses `secrets.GH_PAT` to push commits and create releases. This is set in:
`GitHub repo → Settings → Secrets and variables → Actions → GH_PAT`

The workflow has `permissions: contents: write` which allows it to create releases via the GitHub API.

## 9. Common Issues and Fixes

### Issue: `npm install` gets old AAR despite changing version tag

**Cause:** npm caches GitHub packages locally  
**Fix:**
```bash
Remove-Item -Recurse -Force node_modules\react-native-flutter-chat
npm install github:neerajj911/flutter-chat-sdk#vX.X.X --force
```

### Issue: `git push` rejected — "Updates were rejected because the remote contains work"

**Cause:** CI auto-commits a version bump when it builds a release. Your local branch is behind remote.  
**Fix:**
```bash
git pull --rebase origin main
git push
```

### Issue: Changes not appearing on device even after rebuild

**Cause:** Gradle uses compiled class cache  
**Fix:**
```bash
cd android
.\gradlew clean
cd ..
npx react-native run-android
```

### Issue: Native module not found / app crashes on `openFlutterChat()`

**Cause:** React Native autolinking didn't pick up the module, or `setup.js` didn't run  
**Fix:**
```bash
npm install                    # re-triggers postinstall / setup.js
npx react-native run-android  # forces autolinking rebuild
```

### Issue: Flutter screen is blank / shows nothing

**Cause:** MethodChannel handshake failed — Flutter didn't receive user data  
**Check:** Run with Android Studio → Logcat → filter by `flutter`  
**Cause:** Usually happens if `FlutterChatActivity` isn't registered in AndroidManifest.xml  
**Fix:** Check `setup.js` ran and `AndroidManifest.xml` has `FlutterChatActivity` entry

### Issue: `minSdkVersion` conflict

**Cause:** Your app targets minSdk < 24 but Flutter requires 24+  
**Fix:** In `android/app/build.gradle`:
```groovy
defaultConfig {
    minSdkVersion 24
}
```

### Issue: Build fails — "Gradle version incompatible"

**Cause:** Gradle wrapper version too old or too new  
**Fix:** In `android/gradle/wrapper/gradle-wrapper.properties`:
```
distributionUrl=https\://services.gradle.org/distributions/gradle-8.6-all.zip
```

## 10. Versioning Strategy

### Version Format

Versions follow semantic versioning: `vMAJOR.MINOR.PATCH`

| Part | When to increment |
|------|------------------|
| MAJOR (v**2**.0.0) | Breaking changes — API changed, integration steps changed |
| MINOR (v1.**1**.0) | New features added, backward compatible |
| PATCH (v1.0.**21**) | Bug fixes, UI tweaks, small changes |

### Current Version History

| Tag | What changed |
|-----|-------------|

| v1.0.17 | Various fixes |
| v1.0.18 | Timer/logcat support added |
| v1.0.19 | Timer refinements |
| v1.0.20 | `[ChatScreen]` debug tag added |
| v1.0.21 | "development build" text in AppBar header |

### Install a Specific Version
